前行，无须鲜花

发表于 2017-06-01
字数统计 1,076 | 阅读时长 4

写于五月的最后一天，端午节后一天。

昨晚从舟山回杭州的车上我发了3个小时的呆。发呆这件事从去年开始就变得愈发严重，算是写程序带给自己的副作用，在想清问题的过程中，会顺着思路在头脑里模拟出一系列的相关场景，结果就是发呆的时间越来越长。我自己管这个叫后台运行，在洗澡或者上班的时候会想到特别变态绕来绕去的指令，比如变量未声明即调用会产生什么后果（实际环境中，我们总会把变量直接声明在最前面），这对后来理解程序的运行确实挺有效果。麻烦就是，在想其他事情的时候，想一些往事，往事会越来越清晰。有人说，你能回忆起多远的过去，就能看到多远的未来。这句话我记得特别牢的原因可能是因为听起来像在夸我有远见。但不否认的是，我确实记忆力变好了，这真是件可怕的事情。

尤其是在27岁的时候。

这是一个适合谈婚论嫁的年龄，一个在职场刚摸到点门道准备稳定输出的年纪。这时有什么想法，会被一些人觉得不靠谱，不成熟。你的工作在他们看来觉得还行。就不要去做一些大家都不太理解的事情。招聘的问“你为什么愿意做这项工作，而且比你之前的工资低。”“现在的市场很不好啦，企业能活下来的我看起来都不错，你们企业算是可以的，我就是想不通你为什么会跳槽”为什么？为了钱，为了换一个环境，为了汽车，为了房子。为了结婚，为了生小孩。为了履行我的责任？我回那位对中国实体经济表示深切关注的人：“因为我想，我喜欢我选择的这份工作，我看好这份工作的前景”。好像没有什么说服力，他依然在坚持他的不可理解，事物跳出了他的逻辑，他便开始慌乱了，试图用自己的观点建立一个奇怪的想法。我知道自己不是他所需要的人了。

还好不是，

还好我知道有相信这些的人，我知道那些熬夜码程序的人不单单是为了金钱，我也知道我会有的同类正在哪里，所以我会在这里发博客，我感受得到，从公众号文章，从录制视频教程，从技术博客，从我写下的这句话的内心。一年的时间学习html,css和javascript,还有现在正在用Nodejs搭建的服务器后台。竟也跌跌撞撞走出了一条自己的路。我还记得第一次调试bug成功时满宿舍乱窜的样子，那只是一个英文分号和中文分号的区别，却花了我半小时才找出来。典型的天资愚钝。愚钝却也有愚钝的好，执着多几分，聪明的人也不会做这种对着电脑里的代码傻笑的事情。

反对的声音：

一件事，你去做，就会有人反对。会拿他们的道德准则来框你。我跑步的时候有人会说伤膝盖，你去背单词的时候，他们说用不到。现在是6月一号，儿童节。小时候老师常问：你的梦想是什么？梦想？梦想不是想想的么？梦想的事情难道就要去做么？如果不会后悔那么不做也罢。有赞创始人有句话，很冲。不好听，但是在理“我要干一件想干的事，我怕我也得干，我会想尽办法给自己壮胆，反正他妈的要把这事给干了。老子才30岁，凭什么啊？我至少还能折腾五到十年，不折腾一把老子不服。”
作为不说脏话的孩子，我换了一句作为文章的结尾：
“大圣，此去欲何？踏南天，碎凌霄。若一去不回……”
“便一去不回！”

JS引擎的工作流程

上一章说到函数声明会自动提前，实际就是JS引擎的功劳，电脑CPU只能理解二进制代码，而我们书写的程序需要经过编译之后形成机器码才能输入给CPU执行，这段过程称为编译：包括词法分析-把代码分割成最小化，语法分析-重新组合代码，语义分析-生成语法树，然后是中间代码生成，再经过链接器的链接生成最后的可执行机器码。JS的作为一门解释性语言的差别体现在边编译边执行。而执行编译的便是JS引擎，比如比较著名的谷歌chrome浏览器v8引擎。

踏入面向对象的世界

最近在观看Zepto源码解析的视频时，老师又讲到了$对象与普通对象的区别，还记得《锋利的jquery》一书中说，$对象与普通对象的区别在于，$对象是一个数组结构的对象，但没有数组的push，shift方法。想必在控制台大家也注意这对”[]”。那么它内部是如何实现这个不是数组的对象呢？其实实现这个方法非常容易，也就是这个方法，体现了JS作为一门面向对象编程语言的继承功能——JS原型链。

JS原型英文解析

Every JavaScript object has a prototype. The prototype is also an object.
All JavaScript objects inherit their properties and methods from their prototype.

JavaScript Prototypes
All JavaScript objects inherit the properties and methods from their prototype.
Objects created using an object literal, or with new Object(), inherit from a prototype called Object.prototype.
Objects created with new Date() inherit the Date.prototype.
The Object.prototype is on the top of the prototype chain.
All JavaScript objects (Date, Array, RegExp, Function, ….) inherit from the Object.prototype.
More info: w3schools
首先说下为什么要拉出英文的定义，我在写这篇文档时，查看了前3页的搜索“JS原型链”的文章，其中定义最清晰简洁的还要属w3school，一般来说访问这个网站需要翻墙，国内有一个汉化版，但是不全，比较新的内容没有搬过来，比如英文版的w3school有一个w3.css的小框架，用法跟bootstrap一样，都是往class标签里添加类，用起来比bootstrap更小巧。不过比较冷门。
引用的英文的意思是说，每个JS对象都有一个原型，原型也是对象，JS对象从原型那里继承了属性和方法。既然每个对象都有原型，那肯定有一个顶，也就是原型链最深处的原始原型，下文也说了，即Object.prototype，但，这里有一个常见的误区，就是把原型和对象的prototype属性划等号。在ECMAScript5的语法当中，prototype是函数特有的属性，实例对象的原型则是__proto__,prototype用来存放函数继承给下一代的东西，但是直接访问是访问不到的。例一：
